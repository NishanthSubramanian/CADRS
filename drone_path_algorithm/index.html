CADRS

CADRS (Coordinated & Autonomous Drone Rescue System) is a drone rescue system in which drones are deployed in a specified area and the drones find the best possible path to deliver various payloads like first aid kits, food and clothing to locations that are not easily accessible. They may also be used to deliver a large number of items in a short span of time.

Challenges

  • To get the shortest path from the drone(source) to all the areas of delivery.
  • To be able to move the drones efficiently across the area without collisions and clashes.
  • To get the system of drones to operate in terms of the urgency of the rescue operation.
  • To be able to establish a connection between the drones and a common ground control.

Working

The objective of our drone system is to enable a central server detect urgency calls from distinct areas. This centralized server controls a system of drones.

  1. There are known emergency points where people are in need of supplies, called nodes that the drones have to deliver.
  2. Drones have cameras on them to enable computer vision into these drones.
  3. During their operation of delivering to the known nodes, it may come across new nodes (new people that the drone just recognized during its operation and who need the drone to stop and drop off a few supplies).
  4. The Rescue operation is a time-critical application where continuous connectivity to ground personnel is mandatory. Thus, persistent network connectivity is desirable to propagate information efÔ¨Åciently. Therfore, Wi-Fi cards have been employed in the drone.
  5. Since there's no size constraint on our centralized server, we can employ a Wi-Fi card like IEEE 802.11
  6. The central server recieves the alert from the nodes, deploys the drone that corresponding to that region. The drone follows a path corresponding to the most effecient algorithm for travel and is able to deliver the payload in the least amount of time.

Solution

Our drone system is autonomous and controlled by a centralized server. The two most important aspects to take care of are:

  1. Drone Movement Algorithm
  2. Object Detection

For the drone movement algorithm, we have designed an algorithm called DBPS (Drones' Best Path Algorithm). It dictates which drones are in-charge of which all nodes and in what order do these drones traverse those nodes. Cost of travelling between one node and another is calculated based on two factors: urgency of the destination node and the distance between source node and destination node. For object detection we make use of tensorflow's object detection API which works on images and a python script does all the magic of converting a video stream into frames, passing it to the tensorflow API and with the help of multiprocessing optimize our performace to write the video output into our display screen. Our solution works with real time video streaming as well as video input files to detect objects and output this back into a video showing a demonstration of well the API detects various objects in the video.
